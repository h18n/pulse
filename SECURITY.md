# Security Policy

## Supported Versions
We are currently actively supporting the `main` branch and the latest stable release. Please ensure you are running the most recent code before reporting any issues.

| Version | Supported          |
| ------- | ------------------ |
| v0.1.x  | :white_check_mark: |
| < v0.1.x| :x:                |

## Reporting a Vulnerability

Security is a priority for the Pulse Observability Platform. 

If you discover a security vulnerability, we would appreciate if you could follow these guidelines:
1. **Do not open a public issue.** This gives malicious actors a chance to exploit the vulnerability before we can fix it.
2. Email the core maintainer team at `security@pulse.example.com` (replace with real email when ready) with the details of the vulnerability.
3. In your email, please include:
   - A detailed description of the issue.
   - Steps to reproduce the issue.
   - Potential impact of the vulnerability.

We will try our best to address the issue promptly and keep you updated on our progress. We usually respond within 48 hours to acknowledge the report.
